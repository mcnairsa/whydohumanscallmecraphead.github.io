Exercises

1 of 14 Type

Aka tag name or element selector since they correspond to html elements.

E.g. the following code 

    p {
      color: green;
    }
will set the text colour for all paragraphs (the <p> elements in the html) to the colour green.

2 of 14 Universal

The universal selector '*' selects all elements on a page. E.g. 

    * { 
      font-family: Verdana;
    }

3 of 14 Class

The class selector targets a class attribute in the html and selects all elements that have that class attribute. In the css it is declared by prepending the class name with a full stop. E.g. 

    .title {
        color; teal:
    }

4 of 14 Multiple Classes

Class selectors can be combined in the html to create combinations of effects. E.g. 

	
    <h1 class='title uppercase'>Top Vacation Spots</h1>
	  
    .title {
        color: green;
    }
	
	.uppercase {
		text-transform: uppercase;
	}

5 of 14 ID

Html elements can have an id that is unique to that node in document. In the css the id is selected by prepending the id name with a'#'. E.g. 

  <h1 id="article-title">Titular Article</h1>

  #article-title {
    font-family: cursive;
  }

6 of 14 Attribute

Attributes of an element can be styled. A whole set of wildcard characters can be used to select aspects of the attribute. E.g. 

    <a href="#attribute">Attribute</a>

    a[href*='attribute'] {
      color: lightgreen;
    }

Also '~', '^' and a whole bunch of stuff. See MDN reference for full details of use of selectors on attributes.

7 of 14 Attribute

Apseudo-class is used to change syles based on user actions. E.g., a greyed out Submit button that becomes active when all fields in a form have been filled out. Pseudo-classes include: :hover, :visited, :disabled, :active and many more. See the MDN Pseudo-classes reference for full details.. 

    a:hover {
      color: darkorange;
    }

8 of 14 Classes and IDs

Classes are multi-use so a class ruleset can be used to syles multiple elements. Id's are single-use so id rulesets are used to style a single element.

9 of 14 Specificity

Specificity refers to the relative weight given to css selectors. The more specific selector will override the less specific. E.g. a class selector will override a type selector but be overridden by an id selector.

10 of 14 Chaining

Chaining is linking two or more selectors. Only those elements that to which all selectors apply will be selected. E.g., the following css will set only <h1> elements with a class of special to 69% of the normal font size for the current document. Other <h1> elements and other elements with a class of special will not be styled by this rule. The chained selectors have no space or other character between them.

      h1.special {
        font-size: 69%; 
      }

11 of 14 Descendant Combinator

Descendant combinator selectors are created linking two or more selectors with a space in between. This allows the developer to fine tune the specificity of selectors. Given the following html

    <ul class='main-list'>`
      <li> ... >/li>
      <li> ... >/li>
    </ul>
... we can specify that the <li> elements will be styled with the following rule: 
    .main-list li {
        color: purple; 
      }

12 of 14 Chaining and Specificity

Chaining selectors leads to greater specificity. An element that is covered by one of the parts of the chained ruleset but not the other(s) will not be styled by the unchained, and therefore less specific, rule.

13 of 14 Multiple Selectors

The following code will style both <h5> and <li> the same without the need for separate rule for each. 

      li {
        font-family: monospace;
      }

14 of 14 Review and Quiz Feedback

Note the following code will not work. The <p> element will not display. The style element must contain the css that will be applied to html elements and not html elements themselves. 

    <head>
      <style>
        <p style="color:red;">I'm learning to code!</p>  
      <style>
    </head>
Alternatively, the html element can be styled directly but this is not recommended. See above for how to do it if you really must.
