1 of 7 Introduction

CSS gives the developer control over the styling of otherwise plain html. It controls, inter alia, colours, fonts and position of elements on the page.

2 of 7 CSS Anatomy

Two ways to use CSS are:

  • inline styles that are inline with their associated elements and
  • rulesets that are separate from the html elements being styled.
[For code examples see following sections.]

3 of 7 CSS Inline Styles

CSS can be inserted directly into an html element thus: 

        <p style="color: red;">I'm learning to code!</>
This will be rendered by the browser thus:

I'm learning to code!

Note that inline styles are rarely used in practice ... carry on reading for the right way to do things.

4 of 7 CSS Internal Stylesheet

We can add stylesheets internally within the <head> element of an html file. 

              <head>>
                <style>
                p {
                  color: green;
                }
                </style>
              </head>

5 of 7 CSS External Stylesheet

Or we can put the CSS in a separate file.
The stylesheet itself will look something like this.

              p {
                color: green;
              }

6 of 7 CSS Linking the External Stylesheet

We use the <link> html element to link to our css file from our html file. Like this: 

              <head>
                <link href="./style.css> rel="stylesheet" />
              </head>
Note: the rel="./ ... " path is used when we link to a css file in the same directory [folder] as the html file. This is called a relative path. If the stylesheet is elsewhere on the web we must use the full or absolute path.
Note: the rel="stylesheet" attribute is required so that the browser knows it's a stylesheet.

7 of 7 Review

Seriously? Haven't you had enough? If you really want a review go look at the lesson.

We use external stylesheets, often a single file, in preference to internal stylesheets (a) so that we have a proper separation of concerns and (b) so that we do not have to find all the internal stylesheets, perhaps a large number of them contained in all our html files. This means we have to change the code once and do not risk missing any.
