Position

The available values for the position property of an html element include:

  • static, the default value
  • relative '... to its default position on the web page'. The offset properties top, bottom, left and right determine which way the element will move and by how much. They can be specified in pixels, as a percentage of the relevant dimension of the page or by other measures.
  • absolute takes the element out of the flow of the document so it is ignored by other elements. Its position is determined 'relative to its closest positioned parent element'. Offset properties 'determine the final position'. With absolute positioning the element will still scroll with the rest of the document.
  • fixed positioning fixes the element on the page so that it stays put regardless of user scrolling. We use the offset properties to control where on the page it is fixed.
  • A sticky elements stay where they are while the user scrolls until the scrolling puts it at the bottom of the page then it 'rejoins' the rest of the content and scrolls normally. What position on the page it is stuck to is controlled by the offset properties.

Display

The display property of an element controls whether it can exist on the same line as another or whether it will default to taking up a line all to itself. The possible values of display are:

  • inline which typically puts the element in the normal flow of text. Elements such as <em> that affect the look of a piece of text or elements such as anchors (<a>) and <span> fit in along with the content in their container. inline elements have their width and height properties determined by their container and these properties cannot be adjusted in css. If an element such as a heading (<h1> &c) or <p> have their display property set to a value of inline they will lose their normal behaviour of inserting newlines between themselves and any text or other content before or after them.
  • block-level elements work the other way round. They default to taking up the width of the page. Their height is set by to what they necessary. Both the height and width properties of a block-level element can be set in css.
  • inine-block elements have the best of both worlds. They can appear on the same line as other elements and have their height and width set in css.

z-index

Whether and how elements overlap is controlled by the z-index property. z-index does not work on static elements and has a default value of 0. Changing the default z-index value to an integer greater than 0 will raise that element's priority in the order of visibility. The priority of multiple elements is set by increasing the z-index.

Float

float positions an element within its container. It is 'commony used to wrap text around images.' Flexbox and Grid can be used to get much greater control over the horizontal positioning of elements relative to each other. Typical values of float are left and right.

Clear

clear makes sure that elements do not bump into each other. Possible vlaues for clear are:

  • left
  • right
  • both
  • none
