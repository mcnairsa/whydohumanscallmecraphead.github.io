1 of 16 Introduction

Every html element lives in a box. Knowing how to control the differernt parts of the box is a big part of styling elements. It gives control over borders and the spacing between elements.

2 of 16 The Box Model

The box consists of layers like a series of concentric boxes. The outer layer is the surrounding margin between elements. Within that is a border then padding between the border and the element and the inner content box.

The MDN article 'The box model' has a diagram of the parts of a box. See the rest of the article for a real in-depth view.

3 of 16 Height and Width

An element's default height and width is whatever is needed for the content. This can be overridden with the css height and width properties.

Pixels are an absolute measure of height and width dependent only on the properties of the display device. The element will appear the same size (in pixels) no matter what device it is viewed on.

E.g. the following code 

        #logo {
            height: 30px;
            width: 30px;
          }
will set the size of the element with an id of logo (usually an image) to be 30 pixels high and 30 pixels wide on any device.

4 of 16 Borders

Every element has a border. The default values for the width, style and color properties (which are also the default values for these properties) will render the border invisible but it is still there. If any or all of these values is unspecified then the default values are applied.

The default border is medium none color, where color is the current color of the element. If width, style, or color are not set in the CSS file, the web browser assigns the default value for that property. [Codecademy lesson]
E.g. 
        p.content {
            height: 80px;
            width: 240px;
            border: solid red thick;
          }
or... 
        p.content {
            height: 80px;
            width: 240px;
            border: solid #fff 2px;
          }

5 of 16 Border Radius

The corners of the box may be rounded rather than right-angled. To do this we use the border-radius property. The radius can be set either using pixels or a percentage. 

          container {
              height: 80px;
              width: 240px;
              border-radius: 5px;
            }
Will set a border-radius that is the same rounding as a circle 5 pixcels radius. 
        container {
          height: 200px;
          width: 200px;
          border-radius: 50%;
        }
Will create a perfect circle of 200 pixels radius.

Note that we can also set the radius of individual corners. See the MDN documentation for details.

6 of 16 Padding

The padding property is used to control the space between the content of the box and the border. We can set the four sides to the same value or set them individually with, e.g. border-top.

'The padding property is often used to expand the background color and make the content look less cramped.' (Codecademy)
Note: (from MDN)
Note: Padding creates extra space within an element. In contrast, margin creates extra space around an element.

7 of 16 Padding Shorthand

A shorthand property is used to specify different values for the four sides in one line of code. Thus: 

        p.content-header {
          padding: 6px 11px 4px 9px;
        }

8 of 16 Margin

The margin is the space that separates the element from its neighbors. (MDN) If the margin is the same size all round the box you set the margin like this: 

    p {
      border: 1px solid aquamarine;
      margin: 20px;
    }
Or you can set the top, sides and bottom values specifically: 
    p {
      border: 1px solid aquamarine;
      margin-top: 10px;
      margin-right: 5px;
      margin-bottom: 20px;
      margin-left: 15px;
    }

9 of 16 Margin Shorthand

You don't have to set the four sides on four rows of code. Using margin shorthand you can set them individually in one line. Like this: 

    p {
      border: 1px solid aquamarine;
      margin: 10px 5px 20px 15px;
    }
... which is the same as the previous.

Note: the order is always clock-wise starting with the top: top, right, bottom, left.

Two values sets top and bottom to the first value and uses the second value for the left and right sides. Three values takes the first value for the top, the second for the left and right sides and the third for the bottom.

10 of 16 Auto

If a value of auto is given for one of the margin values it is left to the browser to figure it out. This can be used, for example, to centre an element within its container. Specifying auto in a two-value rule will centre the element horizontally.

Vertical, i.e. left and right margins add together to keep their respective elements apart by the sum of their values. Horizontal (top and bottom) margins do not. Whichever is the greater is taken as the margin between the two elements.

12 of 16 Minimum and Maximum Height and Width

Minimum and maximum height and width help to make sure content stays viewable across a wide range of different screen sizes. E.g.: 

    p {
      color: grey;
      font-size: 16px;
      line-height: 48px;
      margin-top: 60px;
      padding: 10px 20px;
      min-width: 200px;
      max-width: 800px;
      min-height: 200px;
      max-height: 300px
    }

13 of 16 Overflow

The overflow property determines what happens when an element is larger than its container. The default value of visible will make the overflowing content always visible. A value of hidden will hide it and scroll will add scroll bars. E.g.: 

    p {
      overflow: scroll; 
    }

Note: the size of content is not always known at the time of coding the page. If text is inserted into a table cell from a database, for example, the amount of text is determined by what is read from the database, not by the programmer.

14 of 16 Resetting Defaults

Browsers generally have a built-in stylesheet known as the user-agent stylesheet. It is common to reset values on the default stylesheet to created a blank slate. The following 

    * {
      margin: 0;
      padding: 0;
    }
is commonly the first ruleset in a developer-created stylesheet.

15 of 16 Visibility

The visibility property of an element determines whether its content is seen or not. Possible values are: hidden, visible and collapse. The collapse value behaves differently for different elements. See MDN for details.

Note: What’s the difference between display: none and visibility: hidden? An element with display: none will be completely removed from the web page. An element with visibility: hidden, however, will not be visible on the web page, but the space reserved for it will.
[Codecademy]
